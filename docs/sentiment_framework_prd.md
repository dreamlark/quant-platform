## 1. 背景与目标

### 1.1 为什么做

情绪是四源里**最窄、最弱**的一源。现状（`factors/sentiment.py`）只有 4 个**个股级量价代理**，无市场级综合指数，也无资金 / 估值 / 利率 / 衍生品 / 文本任何维度。你提供的框架（经典理论 + 量化方案 + AI 另类）远超现状。补齐后，情绪源可从"个股 frenzy/fear 代理"升级为 **「个股情绪 + 市场综合状态」双轴体系**。

### 1.2 目标

1. 建立**市场级综合情绪指数**（对标中信建投/中信、Baker-Wurgler），可驱动择时观点与看板。
2. 保留并适度扩展**个股情绪**。
3. 接入可选的**资金温度计**与**文本 LLM 情绪**。
4. 全程 analysis-first：情绪产物是"观点 / 状态"，禁止交易动作。

## 6. 指标池与合成方法

**底层指标池（按维度）**

| 维度 | 候选指标 |
|------|----------|
| 量 | 成交金额/总市值比、换手率 |
| 价 | 当日收涨占比、近 N 日收涨占比（宽度） |
| 资金 | 融资净买入额占比、北向净买入、ETF 净申购、主动买入净额（可得时） |
| 估值 | 万得全 A / 申万行业 PE/PB 历史分位 |
| 风险溢价 | 全 A 隐含收益率 − 10Y 国债收益率 |
| 衍生品 | 沪深300 持仓 PCR、隐含波动率（可得时） |
| 文本 | LLM 新闻/股吧情绪得分（负面加权） |

**合成方法（可配置）**

- **主方案（中信建投式）**：每个指标算历史分位数（如滚动 3 年）→ 等权加总 → 综合情绪指数 ∈ [0,100] 百分位。
- **进阶（Baker-Wurgler 式）**：对多变量做 PCA，取第一主成分作为"纯净"情绪；需变量足够多且口径稳定。
- 输出：综合指数 + 子指数（量价 / 资金 / 估值 / 风险溢价），便于下钻与解释。

## 8. 与融合 / 看板的集成

- **个股情绪**：不变，`signal_pool` 中 `sentiment` 分支权重 0.15（可在 `config/settings.yaml` 调）。
- **市场综合指数**：新增 `sentiment_index(date, index_value, sub_volume, sub_money, sub_valuation, sub_riskpremium)` 落库；`monitor` 新增"市场情绪"卡片；`dashboard` 新增情绪温度计可视化。
- **regime 调节（可选进阶，默认关）**：当市场情绪处极端恐惧/贪婪时，对融合**总信号**做置信度缩放（不增减方向，只调强度）。必须经样本外回测验证正面才启用，避免过拟合与结构偏移放大。

## 9. 风险与失效

- **模型失效**：情绪指标被广泛采用后可能失效（你原文已指出）。
- **结构偏移**：投资者结构变化使情绪中枢偏移 → 综合指数用**滚动历史分位**而非固定阈值缓解。
- **数据质量**：融资/北向/ETF 接口偶有缺失或延迟 → 所有新数据源须懒加载 + 缓存 + 缺失降级，不阻断主流水线。
- **前视偏差**：所有指标必须 point-in-time；分位计算只能用截至 t 的历史，禁止未来信息泄漏。

---

## 10. 验收标准

- **数据层**：各指标可拉取并落库，具备缓存与降级；缺失不阻断主流水线。
- **指数**：综合指数与子指数计算正确，历史分位口径一致、无前视。
- **集成**：`monitor` / `dashboard` 正确展示市场情绪与温度计；**个股融合结果不变**（回归校验）。
- **择时（T2）**：温度计阈值信号可输出，并附样本外回测（年化/回撤/超额）。
- **红线**：全程无交易/下单代码路径；LLM 文本情绪遵守合规口径（分析观点，非买卖建议）。
